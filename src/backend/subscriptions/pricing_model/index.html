Each meaningfully moves the numbers above

✓ shipped Unified parse cache

OneClick and Registry now share one cache keyed on the canonical contract text. The extraction call — the biggest of OneClick's gpt-5.2 calls — is served from cache on any repeat or cross-tool run, and a contract parsed in Add Work makes OneClick's first calc a hit.

Warm runs drop 3 → 2 gpt-5.2 calls (single), ≈ −⅓ to −40% each. Drag cache hit rate to see the blended steady state.

next win Cache or downgrade the advisory passes

With extraction cached, basis-detection + split-verification are now the entire OneClick bill on a warm run — 2 gpt-5.2 calls that re-send the full contract every time and are log-only / advisory (never change a payout).

Cache them by the same text hash, gate them behind a flag, or move them to a mini tier → a warm OneClick run approaches free.

up to −90% input Turn on prompt caching

The advisory passes (and Zoe's answer + page-citation calls) still re-send the same contract text uncached.

Cached input is $0.175 vs $1.75/1M on gpt-5.2 — reusing the contract prefix cuts most of the remaining input bill.

uncapped Cap reasoning output

OneClick's gpt-5.2 calls leave max_completion_tokens unset. On a reasoning model, output = hidden reasoning + JSON, billed at $14/1M and effectively unbounded.

Set a cap / lower reasoning effort — output is the single biggest driver on the calls that remain.
